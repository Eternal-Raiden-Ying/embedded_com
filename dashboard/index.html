SC171 环境说明

板端端侧环境配备了 Qualcomm 专用 QNN 推理芯片。视觉检测在 aarch64 底层运行时会自动调配 CPU/GPU/NPU 通道进行运算加速，Orchestrator 内部提供 safety-gate 逻辑来约束运行时的异常状态转移。

运行启动与守护时序调试

详细展示多进程启动校验、套接字端口及常见故障急停排障方法。

Full 模式启动检查链

检查 1
清理 stale sockets 套接字残余
rm -f /tmp/robot_stack/*.sock，初始化套接字权限。
检查 2
拉起 VISTA 感知服务端
使用 /usr/bin/python3 -m vision_module.app.app，后台建立摄像头获取。
检查 3
轮询等待 vision_req.sock 产生
等待进程在端口正常监听，最多自检 35 秒。
检查 4
拉起 Orchestrator 控制状态机
打开底层物理串口 /dev/ttyHS1。
检查 5
轮询等待 task_cmd.sock 与 vision_obs.sock 建立
验证多套接字之间读写管道联通。
检查 6
开启 Mobile Gateway 小程序桥接网关
前置鉴权，MQTT 离线检测机制启动。
就绪
进入 IDLE 待命模式
自检成功，系统进入热待机，可以接收小程序端指令。

UDS socket 套接字详细定位

/tmp/robot_stack/vision_req.sock

Orchestrator 调配 VISTA Perceptual Stage 感知阶段转换的请求端点。

/tmp/robot_stack/task_cmd.sock

Mobile Gateway 转发用户 fetch_object 指令到状态机的控制输入端点。

/tmp/robot_stack/vision_obs.sock

VISTA 高频推送 obs_class="control" 距离角度控制包给状态机的对齐决策端点。

/tmp/robot_stack/task_ack.sock

状态机执行任务终点或故障超时时，反向推送异步返回包通知网关的反馈端点。
